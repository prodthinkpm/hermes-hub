# Agent / Profile 管理需求文档
版本：v0.1  
日期：2026-05-18  
状态：草案（待对齐）

## 1. 背景
Hermes Hub 当前已经具备基础 Agent 列表、详情、日志、SOUL/config 编辑等能力，但在多 Profile 生命周期管理上仍存在体验和一致性问题，主要体现在：

1. 创建、重命名、删除入口分散。  
2. 部分操作安全边界不一致（是否依赖 Hermes CLI）。  
3. 失败原因可见性不足（如 CLI 不存在、端口冲突）。  

本需求文档聚焦“多 Profile 新增与编辑”。

## 2. 目标
1. 提供一致的 Agent 生命周期操作：Create / Rename / Delete。  
2. 所有高风险变更遵循“显式确认 + 审计友好”的交互原则。  
3. 所有关键操作失败都能给出可理解错误信息。  
4. UI 以高频操作效率为优先：列表直达，详情页以配置/状态查看为主。

## 3. 非目标
1. 不替代 Hermes 原生 setup 全流程。  
2. 不实现远程 shell 任意执行。  
3. 不在本阶段引入复杂 RBAC 多租户模型。  
4. 不在本阶段实现批量重命名/批量删除。

## 4. 用户场景
1. 作为运维，我要快速创建一个新 Agent，并马上看到下一步 setup 命令。  
2. 作为运维，我要在列表里直接重命名 Agent，不进入详情页。  
3. 作为运维，我要在列表里删除 profile 类型 Agent，并有二次确认保护。  
4. 作为运维，我在操作失败时需要知道“为什么失败、怎么修复”。

## 5. 功能需求
### 5.1 Create Agent
1. 支持模式：`blank` / `clone` / `clone-all`。  
2. 支持可选 `clone-from`。  
3. 支持可选 `--no-alias`。  
4. 创建成功后返回建议下一步（setup/chat）。

### 5.2 Rename Agent
1. 入口在列表操作列。  
2. 通过应用内弹窗输入新名称。  
3. 名称校验：仅字母、数字、`-`、`_`。  
4. 成功后刷新列表并提示。

### 5.3 Delete Agent
1. 入口在列表操作列。  
2. 仅 `type=profile` 可删除，`type=default` 禁止删除。  
3. 二次确认：确认弹窗 + 名称匹配输入。  
4. 成功后刷新列表并提示。

### 5.4 详情页职责
1. 不承载 Rename/Delete 管理入口。  
2. 保留状态查看、config/SOUL/skills 相关内容。  
3. config 采用“可编辑关键项 + raw 查看”的双视图。

## 6. 安全与一致性要求
1. Rename/Delete/Create 均应走 Hermes CLI 语义，不允许一部分走 CLI、一部分直接改文件系统。  
2. 删除操作必须有前端二次确认与后端类型校验双保险。  
3. 任何 CLI 依赖失败（如找不到 `hermes`）必须返回明确修复建议。

## 7. 可观测性要求
1. 关键操作失败要包含：执行命令、失败原因摘要。  
2. 启动失败要显式提示端口冲突。  
3. 对用户暴露可读错误，不暴露敏感信息。

## 8. 验收标准
1. 在列表页可完成 Rename/Delete 主流程。  
2. `default` 类型 Agent 不可删除。  
3. CLI 缺失时操作失败文案可指导修复。  
4. 详情页不再显示 Agent Management 区。  
5. 不改代码逻辑之外的无关页面行为。

## 9. 待确认项
1. Delete 是否需要额外“冷静期”开关（例如 3 秒倒计时）。  
2. 是否在 Settings 增加 `HERMES_BIN` 可视化配置。  
3. Rename 成功后是否需要自动跳转到详情页。
